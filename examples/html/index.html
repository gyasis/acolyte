What it is

DSPy (Declarative Self-improving Python) is a Stanford framework for programming language models instead of prompting them. You declare a Signature (inputs → outputs), wrap it in a Module (Predict, ChainOfThought, ReAct), and let an Optimizer tune the prompt for you against a metric.

Why it matters

Raw f-string prompts age badly. Modules are swappable. Optimizers like MIPROv2 and GEPA replace hand-tuning with measurable improvement loops.

Three primitives to know

Signature declares the contract — what fields go in, what fields come out, plus a one-line description. The model never sees it directly; the adapter renders it into provider-specific messages. Module wraps the signature with a calling strategy: Predict is a single LLM call, ChainOfThought prepends a hidden reasoning field, ReAct loops the model with tool calls until done. Optimizer takes a module + a metric + labeled examples and produces a tuned version with better prompts and demos.
