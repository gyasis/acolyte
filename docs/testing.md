# Testing

How we verify acolyte works end-to-end. Two layers:

| Layer | Lives in | Purpose |
|---|---|---|
| **Unit / build** | `acolyte/` | Type checks, build succeeds, dist is valid |
| **Live page** | A separate "test page" repo | Real LLM call, real RAG, real DOM, real voice |

We deliberately do not bake a hosted demo page into this repo — the
widget is supposed to work in any page, so the test runs against an
unrelated page that is NOT acolyte's own.

## The current E2E test setup

> Status: **wired up and verified** as of the first migration pass.
> See "Migration history" below.

### Topology

```
  ┌───────────────────────────────────┐
  │  Browser (Chrome / Firefox / …)   │
  └─────────────┬─────────────────────┘
                │  http://127.0.0.1:8766/index.html
                ▼
  ┌───────────────────────────────────┐
  │  python3 -m http.server :8766     │
  │  serves dspy-course/              │
  │                                   │
  │  dspy-course/                     │
  │  ├─ index.html (built by build.sh)│
  │  ├─ acolyte.config.json           │   ← all widget config here
  │  ├─ acolyte/  ──────────┐         │   ← SYMLINK to acolyte/dist
  │  ├─ skin-claude.css     │         │
  │  └─ …                   │         │
  └────────────────────────┼──────────┘
                           │
                           ▼
                  ~/Documents/code/acolyte/dist/
                  (built by `npm run build`)

  ┌───────────────────────────────────┐
  │  python3 cors-proxy.py :8767      │
  │  reads .env, forwards /api/* to   │
  │  $OLLAMA_UPSTREAM, serves         │
  │  /chat-config with env-loaded keys│
  └─────────────┬─────────────────────┘
                │
                ▼
       (your real Ollama box on the LAN)
```

### Why this topology

| Choice | Reason |
|---|---|
| Test page is a **real, unrelated** site (DSPy course generated by another skill) | Proves the widget works on a page it does not own. The course's structure, classes, headings, and skin are not acolyte's. |
| Bundle is **symlinked**, not copied | `cd acolyte && npm run build` → next browser refresh of the course picks up the new bundle. No drift, no copy-paste, no version skew. |
| Config is **JSON**, not inline | Editing widget behavior never touches the host page's HTML. Matches the plug-and-play promise. |
| Server runs on **127.0.0.1**, not 0.0.0.0 | We're testing locally — no need to expose the test pages to the LAN. |
| Course skin (`claude`) stays intact | The skin loader was untouched by the migration. Acolyte's accent (`#CC785C`) is configured to match terracotta, so the chat chrome harmonizes with the course visually. |

### Two-command bring-up

```bash
# Terminal 1: serve the test page
cd ~/Documents/code/SIO/dspy-course
python3 -m http.server 8766 --bind 127.0.0.1

# Terminal 2: serve the Ollama proxy (only needed if Ollama is remote)
cd ~/Documents/code/acolyte/server      # or cp .env from dspy-course
python3 cors-proxy.py

# open the page
xdg-open http://127.0.0.1:8766/index.html
```

If your Ollama is on `localhost:11434` with `OLLAMA_ORIGINS=*`, you can
skip the proxy entirely — edit `dspy-course/acolyte.config.json` so
`llm.host` is `http://localhost:11434` directly.

## Smoke test checklist

After opening the page, verify:

- [ ] Page loads with **Claude skin** (parchment background, terracotta accent)
- [ ] Course content renders normally (modules, headings, code blocks)
- [ ] **💬 button** appears bottom-right
- [ ] Click it → panel slides in from the right
- [ ] Status pill in the header reads `Ollama · <model-name>` (green)
- [ ] Teacher persona greeting visible in the message area
- [ ] Type a question (e.g. *"How does ChainOfThought differ from ReAct?"*)
- [ ] Streaming response appears token-by-token in the bubble
- [ ] Markdown renders (code blocks, lists, **bold**)
- [ ] If math: `$…$` renders as KaTeX, not raw backslashes
- [ ] `Ctrl+K` toggles the panel
- [ ] `Esc` closes the panel
- [ ] FAB stays visible after closing — single click reopens

If any of these fail, capture the browser console + the failing step and
file it — that's the bug list before we declare the migration done.

## Cycle for iterating

```bash
# 1. Edit acolyte source
$EDITOR ~/Documents/code/acolyte/src/widget.ts

# 2. Rebuild — same shell or a watcher
cd ~/Documents/code/acolyte && npm run build
# OR: npm run dev   (tsup --watch)

# 3. Refresh the browser
# The course's <script src="acolyte/index.js"> resolves through the symlink
# to the freshly built bundle.
```

No course rebuild needed for widget-only changes. If you change anything
in `dspy-course/_base.html`, `dspy-course/modules/*`, or any course
content, run `bash dspy-course/build.sh` to regenerate `index.html`.

## What we explicitly do NOT test in this repo

- Real cloud API costs — that's the user's BYOK problem
- Real Ollama LAN connectivity — that's the user's infrastructure
- Real voice playback — depends on OS / browser voices
- The DSPy course's *content* — that lives in a different project for a
  different purpose

## Future: automated tests

Not yet built. When they land they will live in `tests/` and use:

- `vitest` — unit tests on internal modules (RAG ranking, cache hashing,
  TTS markdown stripping, data-attribute coercion, deep-merge)
- `playwright` — browser integration tests against `examples/html/`
  with a mock provider, verifying the smoke-test checklist programmatically

Adding these requires committing to a test runner — see the constitution
rule on new dev deps.
